开启了畅言评论的支持， Coding 的跳转继续审核中

发表于 2018-04-15 | 分类于 hexo | | 阅读次数:

周五的时候给 Hexo 博客的评论模块 Gitment 变更为了畅言，主要是 Gitment 的样式真心太丑了，并且能还发现很多站长们竟然都没有 GitHub 账号，无语了都！ Gitment 好像也没有“回复”评论的功能，所有评论竟然都是一个样，这个明显接受不了呀！看了不少 Hexo 博客站长都比较推荐畅言，我也就去把N年前的畅言账号给从新设置恢复使用了，今天好像新的 Hexo 博客站点也通过了审核，可以正常的使用评论了。

自从周五将 Hexo 博客托管到 Coding 以后，国内的访问速度是稳定了，但是就是被加上了一个“跳转”比较讨厌， Coding 后天提示只要放上 Coding 的链接后通过审核后即可i取消这个“跳转”（首次访问），悲催的是我是周五开始托管到 Coding ，周六日非工作日不审核，郁闷死了都。

昨天晚上明月研究了一下 Vim 下编辑 MarkDown 文档的增强配置【利用Vundle给Vim安装插件支持 MarkDown 语法、实时预览等】，今天正好发文体验一下，还是有点儿不习惯，但是有了 MarkDown 语法高亮后，撰写 MarkDown 文章的时候， MarkDown 语法标记就很清晰的高亮显示出来了，体验还是很不错的，就是对nerdtree插件还是不习惯，以后多发文章应该就习惯了。

自今天开始，以后 Hexo 博客发布的文章我都会尽量的带上文章缩略图， NexT 主题竟然支持文章缩略图，威武呀！这个主题真的是太强大了！赞一个！

您的支持将鼓励我继续创作！
